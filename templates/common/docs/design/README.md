# 功能技术设计目录

- 按版本存放：`vX.Y.Z/<需求名>.md`。
- 输入是 `docs/requirements/.../final/` 的需求定稿和 `docs/architecture/` 项目基线。
- S 级需求可不生成完整设计；M/L 级按 `.ai/skills/feature-design.md` 执行。
- 只记录 how 和相对项目基线的增量，不复述已确认的需求或整份架构基线。

## 设计结构

1. 目标与约束引用
2. 风险等级与理由
3. 现状及影响范围
4. 模块、职责与依赖
5. 技术接口契约
6. 数据模型增量
7. 核心流程与状态转换
8. 错误、并发、幂等与兼容
9. 质量属性增量
10. 部署、迁移与回滚
11. 测试与验证
12. 假设和开放问题
13. 交付验证（ready/conditional/not-ready、证据与阻塞项）
